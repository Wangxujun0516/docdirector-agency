# Maintenance Sweep Workflow

**Trigger**: Code changes detected, support ticket spike, or scheduled.

## Process
1. **Maintenance Agent** identifies changed components using `freshness-monitoring`.
2. **Research Agent** pulls latest context.
3. **Reviewer Agent** assesses current docs vs. reality.
4. **Drafter + others** propose targeted updates.
5. **Human Director** approves changes.
6. **Analytics Agent** suggests broader improvements.

This keeps documentation alive without constant manual effort.
